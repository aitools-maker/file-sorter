# 🗂 ファイル仕分けアプリ（File Sorter）

散らかった Mac のファイルを、**1 つずつプレビューしながら、提示された移動先候補ボタンをクリックして整理先フォルダへ移動**していくためのローカル Web アプリです。全自動で勝手に動かさず、**最後は人が 1 個ずつ判断**する方式。依存ライブラリなし（Python 3 の標準ライブラリのみ）・**完全にローカル**（外部に何も送信しません）。

> macOS 向け。`config.json` を編集すれば、自分のフォルダ構成に合わせて使えます。

---

## クイックスタート

1. このフォルダをダウンロード／クローンする。
2. `config.json` を開いて **`root_dir`（整理先フォルダ）** と **`main_folders`** を自分の環境に合わせる（後述）。そのままでも `~/Documents` 整理用のデフォルトで動きます。
3. 起動する：
   - **`start.command` をダブルクリック**（初回は下記「初回の許可」を参照）
   - または ターミナルで：
     ```
     cd <このフォルダ>
     python3 server.py
     ```
4. ブラウザで http://localhost:8765 が自動で開きます。停止はターミナル窓で `Ctrl + C`。

### 初回の許可（macOS）

- **Gatekeeper**：`start.command` を初めて開くと「開発元を確認できません」と出ることがあります。**Finder で右クリック → 開く**（または「システム設定 →プライバシーとセキュリティ」で許可）。
- **Python**：入っていない場合は、ターミナルで `xcode-select --install`（Command Line Tools）を入れるか、[python.org](https://www.python.org/) から Python 3 を入れてください。
- **フォルダへのアクセス権**：デスクトップ／書類／ダウンロード等を読むため、初回に macOS の許可ダイアログが出たら「許可」を選んでください。

## 設定（`config.json`）

`server.py` と同じフォルダの `config.json` を編集します。`config.json` が無ければ汎用デフォルトで起動します。`_` で始まるキーはコメント扱いで無視されます。

| キー | 意味 |
|---|---|
| `root_dir` | **整理先のルートフォルダ。移動先候補はこの配下のみ提示される。** `~` はホーム |
| `extra_scan_roots` | 追加で「片付ける対象」にする場所（移動先候補としては出さない）。`[表示名, ホーム相対 or 絶対パス]`。存在する物だけ有効 |
| `main_folders` | `root_dir` 直下の主要フォルダ（トップ扱い） |
| `keyword_rules` | `[正規表現, 推奨移動先]`。ファイル名・現在地・拡張子に一致すると候補上位に |
| `ext_rules` | `[拡張子グループ名, [推奨移動先...]]`。グループ名: `IMAGE/VIDEO/AUDIO/DOC/SHEET/SLIDE/PDF/TEXT/DTM` |
| `kind_home_tops` | 種別の「定位置トップ」。ここ以外に在ると「怪しい順」で上位に来る |
| `stable_atomic_prefixes` | 何があっても 1 個の塊として扱う（中を個別に仕分けしない）サブツリー |
| `exclude_name_substr` / `exclude_name_prefix` | 仕分け対象・候補先から外すフォルダ名 |

`server.py` 冒頭の `FOLDER_UNIT_THRESHOLD`（既定 25）を超えるファイルを含むサブフォルダは「確立済みフォルダ」として塊扱いになります。

## 使い方

1. 左の**キュー**からファイルを選ぶ（既定は「怪しい順」＝拡張子と現在地が食い違う物が上）。
2. 中央に**プレビュー**（画像 / 動画 / 音声 / PDF / テキスト / `.docx` / `.pptx` / `.xlsx` / `.csv` `.tsv`）と現在地が出る。
   - Office 系（Word/PowerPoint/Excel）は外部ライブラリなしで本文テキストだけ抽出して表示（書式・画像は出ない）。旧 `.doc` `.xls` は非対応。
   - iWork（Keynote `.key` / Pages `.pages` / Numbers `.numbers`）はファイル内に埋め込まれた**プレビュー画像**を表示。
   - 画像は**ホイールでズーム / ドラッグでパン / ダブルクリックで原寸**。テキスト等は `Ctrl/⌘+ホイール` で文字サイズ変更。
3. 下の**移動先候補ボタン**をクリック → そのフォルダへ即移動（一致理由バッジ付き）。
   - 候補に無ければ**入力欄**にパスを打って「この場所へ移動」（無ければ自動作成）。
   - 今の場所で良ければ **✓ ここで良い（確認済み）**、不要なら **🗑 ゴミ箱へ**。
4. フォルダ（📁）はクリックで中を開いて掘れる。「🔬 さらに詳しく仕分け」で配下の全ファイルを 1 つずつ並べられる。

### キーボード操作

| キー | 動作 |
|---|---|
| `1`〜`9` | その番号の候補へ移動 |
| `K` | ここで良い（確認済み） |
| `S` | スキップ |
| `R` | 名前を変更 |
| `D` | ゴミ箱へ移動 |
| `U` | 直前の操作を元に戻す |
| `↑` `↓` | 選択を上下 |

## 安全設計

- **完全消去はしない。** 操作は「移動(mv)」か「ゴミ箱(`~/.Trash`)へ移動」のみ。`os.remove` / `rmtree` は使いません。
- 移動先が無ければ作成。**同名衝突時は ` (2)` を付けて退避**（上書きしない）。
- 全操作を `moves.jsonl` に追記し、**`U`（元に戻す）** で巻き戻せます。
- アクセス・移動先は **`root_dir` と `extra_scan_roots` の範囲内のみ**（その外＝システム領域などへは移動・アクセス不可）。
- 写真／ミュージック等の **管理ライブラリ（`.photoslibrary`・`.musiclibrary` 等）は一覧から除外**（macOS が保護していて移動できないため）。
- `.logicx` `.key` 等の macOS パッケージ、ファイル数が多い確立済みフォルダは「1 個の塊」として扱い、中身は展開しません。
